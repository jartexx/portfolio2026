Approach

How I Design

My process adapts to each project, but the foundations stay consistent.

◇ 01

Discover

Stakeholder interviews, competitive analysis, data audits. Understand the constraints before proposing solutions.

◇ 02

Define

Information architecture, user flows, service blueprints. Translate research into clear strategic direction.

◇ 03

Design

Design systems, component libraries, high-fidelity prototypes. Build modular solutions, not one-off screens.

◇ 04

Validate

Usability testing, stakeholder reviews, data analysis. Iterate based on evidence, not assumptions.

◇ 05

Ship & Iterate

Launch, monitor metrics, refine. Design doesn't end at handoff.

CASE STUDY 01

E-Commerce Platform Migration

Migrating an e-commerce operation from Magento to Shopify and turning a "traffic loss" into a profitability win.

RoleLead UX Strategist & Architect
Timeline8 months
Year2024-2025
TeamCross-functional, 12 people

The Challenge

Curacao's legacy Magento platform had become a growth bottleneck. The monolithic architecture meant even minor UI updates required developer intervention.

But the deeper problem? The Information Architecture mirrored the company's org chart, not the customer's mental model.

Before/after site map comparison

What I Did

I reframed the project from a "redesign" to a re-platforming strategy — treating it as an operations transformation.

  1. Audit & Benchmark — Mapped the operational friction in the legacy stack.
  2. IA Restructuring — Rebuilt navigation based on purchasing behaviors, not inventory taxonomy.
  3. Component-Based Design System — Created a modular UI library for Shopify.
Component library & design system overview

The Hard Parts

Scope creep: Maintaining feature parity with a decade of custom Magento features.

Culture shift: Moving stakeholders from "Dev-led updates" to "Design-system-led updates."

The Results

MetricBeforeAfterChange
Bounce Rate42%32.67%-22%
ROAS (Q4)$21$29.18+39% YoY
Revenue (Oct)Baseline+6.6% YoY↑ Despite lower traffic
Dev dependencyHighNear-zeroTeam autonomy

The Narrative: Even though traffic dropped 38% post-migration it was expected due to re-platforming. Revenue still grew, ROAS hit all-time highs. We stopped buying window shoppers and started converting buyers.

Artifacts: Site Maps · Migration Strategy Deck · Component Library · Checkout Flow Prototypes

CASE STUDY 02

Miden Crypto Wallet

Designing a privacy-first crypto wallet that onboards users through their messaging platforms.

RoleLead Product Designer
Timeline4 months
TeamCore team of 5

The Challenge

Miden is a privacy-first blockchain built on Zero-Knowledge proofs. The technology is revolutionary — but the UX has historically been hostile.

The Insight

The Privacy Paradox: Users say they value privacy, but choose convenience. The wallet should ride existing "social rails" — encrypted messaging apps users already trust.

Competitive analysis — 5 wallet comparison

What I Did

  1. Competitive Analysis — Audited 5 wallets to map friction points.
  2. Mental Model Mapping — Reframed "Miden Notes" as text-message analogues.
  3. Network Effect Design — Users first receive funds via chat link, then set up the wallet.
Share-to-wallet onboarding flow

The Result

A "Social Wallet" vision integrating note dispatching into the OS share sheet. Every messaging app becomes a crypto distribution channel.

Artifacts: Competitive Analysis · Share-to-Wallet Flows · Dark Mode UI System · Toggle Prototypes
